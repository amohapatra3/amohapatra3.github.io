Documentation samples

At AWS, I produced documentation across multiple channels such as developer guides, API and CloudFormation references, and UI text. The following are links to some samples of my work in PDF form taken directly from the public AWS docs websites for Amazon ECS and AWS Cloud Map:

  • Example CFN templates for Amazon ECS: Created and tested templates for various ECS resource types. Templates can be used by customers to get a sense of how they can use Infrastructure-as-code (IaC) to create ECS task definitions, services, clusters etc. Sought SME feedback and leveraged generative AI wherever it made sense. Amazon Q CLI was my tool of choice for generative AI.
  • Refactored AWS Cloud Map documentation to be more scenario-based: Followed the maxim of 'one scenario per page' on most pages in the guide. This specific example refers to the Cloud Map services documentation and includes conceptual topics (what is a service, what are the supported health check and DNS configurations) and CRUD procedures.
  • Cross-service integration chapter including multiple content types. This example includes a table for various storage options for ECS tasks, and the documentation for EBS volumes on ECS that includes example configurations, conceptual considerations, and troubleshooting information.
  • Open-source project I contributed to that provides customers CLI-based getting started tutorials and instructions for generating further tutorials.: Before repo creation, I helped refine the prompts and tutorials through testing. I also incorporated/encouraged incorporation of some of the tutorials in our developer guides during the first phase of the project. As part of repo creation during the second phase, I tested, updated, and then committed tutorials and scripts. The repo provides a CLI experience for customers that may not be able to use a console.
